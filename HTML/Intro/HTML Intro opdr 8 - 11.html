Semantics

9. Sinds de komst van HTML5 zijn er elementen die betekenissen hebben. Wat wordt er bedoeld met HTML semantics? Waarom is het gebruik van semantics belangrijk?

HTML semantics vertellen je wat dingen betekenen.

Het gebruik van semantics vergroot de leesbaarheid van je code. Je hoeft minder css te gebruiken.

Wat voor content zou je in de volgende elementen gebruiken? Geef in je antwoord voor elk element een toelichting.

  • <header>

    • Het <header> element een container voor inleidende inhoud of een reeks navigatielinks.

  • <footer>

    • Het <footer> element definieert een voettekst voor een document of sectie.

  • <main>

    • Een <main> element representeert de hoofdinhoud op de webpagina

  • <nav>

    • Het <nav> element (HTML Navigation Element) representeert een sectie van een webpagina dat links bevat naar andere pagina’s of naar onderdelen binnen deze webpagin

  • <section>

    • Een section element definieert een sectie in een web-document

  • <aside>

    • Een <aside> element is verbonden met de inhoud die het ondersteunt. Het is bijvoorbeeld gerelateerd met een post of artikel. Het kan bijvoorbeeld links bevatten die geralateerd zijn aan het artikel.

  • <article>

    • Een <article> representeert een component van een webpagina dat bestaat uit een vrijstaande compositie in een document, pagina, applicatie of site.

headers

11. Onderzoek en leg uit waarom headers zowel belangrijk zijn voor zoekmachines als voor de gebruikers.

Elk bericht en elke websitepagina maakt gebruik van titels of koppen. Deze kun je als lezer gemakkelijk herkennen. Maar zonder hiërarchisch onderscheid, herkent een zoekmachine de titels niet. En is het moeilijk voor een zoekmachine op te bepalen waar de tekst over gaat. Daarom is het belangrijk om je koppen te labelen als headings met daarbij het juiste hiërarchische niveau.
